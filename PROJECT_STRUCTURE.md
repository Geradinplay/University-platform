# Структура проекта University-platform

## Описание
Проект реорганизован для лучшей масштабируемости и поддержки. Файлы распределены по смыслу функциональности в соответствующие папки.

## Назначение папок

### `api/`
Содержит все модули, отвечающие за взаимодействие с серверной частью и управление данными. Включает реальный API клиент и mock версию для разработки.

### `public/`
Статические файлы, доступные в браузере. HTML страницы и глобальные стили приложения.

### `src/js/auth/`
Модули, отвечающие за аутентификацию пользователей - вход в систему и регистрацию.

### `src/js/handlers/`
Обработчики событий DOM - drag & drop, контекстные меню, обработка форм.

### `src/js/utils/`
Вспомогательные функции и утилиты, используемые в других частях приложения.

### `src/js/components/`
Компоненты пользовательского интерфейса - переиспользуемые части, такие как менеджер задач.

### `src/js/models/`
Модели данных приложения - классы и структуры для представления сущностей (например, Карточка урока).

### `src/css/`
Структурированные стили CSS, разделённые по базовым стилям, макету и компонентам для удобства поддержки.

## Преимущества новой структуры

✅ **Модульность** - каждый файл имеет чёткое предназначение
✅ **Масштабируемость** - легко добавлять новые компоненты и модули
✅ **Поддерживаемость** - логическое разделение кода упрощает навигацию
✅ **Переиспользуемость** - компоненты и утилиты расположены отдельно
✅ **Организованность** - сходные функции группируются в папках

## Импорты файлов

При импорте файлов используйте относительные пути от корня проекта:

```javascript
// Из файла в src/js/main.js
import TaskManager from './components/TaskManager.js';
import { utils } from './utils/utils.js';
import api from '../../api/api.js';
```
